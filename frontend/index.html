GameLens API Endpoints

When the backend is running (e.g., via Docker), the API is typically available at http://localhost:8000.

Health Check

GET /health

Checks the health status of the API. Returns a simple status message.

Example: curl http://localhost:8000/health

User Profile

GET /user/{steam_id}/profile

Retrieves a user's profile information and their owned games. If the user is not found in the database, it attempts to fetch data from a simulated Steam API and stores it.

Example: curl http://localhost:8000/user/123456789/profile (replace {steam_id} with an actual ID)

User Game Categories

GET /user/{steam_id}/categories

Retrieves games owned by the user, categorized by genre. Returns a 404 if the user is not found.

Example: curl http://localhost:8000/user/123456789/categories

Game Deals

GET /deals/{game_name}

Fetches the latest deals for a given game name. It first checks the database and, if no deals are found, queries a simulated scraper and stores new deals.

Example: curl http://localhost:8000/deals/Cyberpunk%202077 (URL encode game names with spaces)

User Recommendations

GET /user/{steam_id}/recommendations

Provides game deal recommendations for a user based on their played game genres. Returns a 404 if the user is not found.

Example: curl http://localhost:8000/user/123456789/recommendations
